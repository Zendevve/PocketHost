## v1 Requirements

### Core Architecture

- [x] **CORE-01**: User can spawn and manage the Java Minecraft Server natively on Android.
  - **Outcome:** Implemented in Phase 1 via `serverManager.ts` bridging to native `ServerProcessModule`. Downloads handled, server boots from UI.
- [x] **CORE-02**: User can minimize the app without the server being abruptly killed by Android (Background Service).
  - **Outcome:** Achieved via Android Foreground Service; confirmed by Phase 1 success criteria: minimizing does not suspend/kill Java process.
- [x] **CORE-03**: User can cleanly stop the server process.
  - **Outcome:** Implemented stop action via serverManager; logs indicate clean shutdown ("Saving chunks", "Server Stopped"). Not explicitly unit-tested but exercised in UAT.

### Dashboard Core

- [x] **DASH-01**: User can view live console logs (stdout/stderr) from the server.
  - **Outcome:** Console.tsx renders capped log stream from Zustand store updated by native events; confirmed working.
- [x] **DASH-02**: User can configure basic server startup parameters (e.g., Memory limit).
  - **Outcome:** Implemented in Phase 3 via memory slider UI (up to 4GB) mapped to `-Xmx` JVM flag.
- [x] **DASH-03**: User can configure the target world and easily switch active worlds.
  - **Outcome:** Implemented in Phase 3 via world path input; creates/switches world folders as expected.

### Plugin Management

- [x] **PLUG-01**: User can easily toggle and install `.jar` plugins for their server instance.
  - **Outcome:** Implemented in Phase 4: .jar import via expo-document-picker; plugins listed with enable/disable toggles.
- [x] **PLUG-02**: User can manage existing plugin configurations.
  - **Outcome:** Implemented in Phase 4: YAML config discovery, read/write via js-yaml, in-place editor, and reload command.

### External Networking

- [x] **NET-01**: User can expose the Android server to the internet using Playit.gg tunnels overriding CGNAT.
  - **Outcome:** Achieved in Phase 2: playit-plugin.jar auto-injected; tunnel allocation from logs; claim flow exposed.
- [x] **NET-02**: User receives a copyable remote join-address when the server boots successfully.
  - **Outcome:** UI displays public `ip:port` after tunnel allocation; claim link shown before activation.
