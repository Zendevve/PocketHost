# Milestone v1.1: Backup & Polish

**Status:** ✅ SHIPPED 2026-04-28
**Phases:** 5-7
**Total Plans:** 3

## Overview

Desktop-grade admin tools for mobile Minecraft server hosting: world backup/restore system, nested YAML config editor, and plugin metadata extraction. All built on the v1.0 stable backend.

## Phases

### Phase 5: Backup Foundation

**Goal:** Users can backup and restore their server worlds with safety guarantees.
**Depends on:** Phase 1 (server lifecycle), Phase 2 (FileSystem/store patterns)
**Plans:** 1 plan

- [x] 05-01: Backup and Restore System

**Details:**
- ZIP backup creation using adm-zip with base64 file reads
- Backup history persisted via Zustand/AsyncStorage
- Dual-confirmation restore (dialog + world name typing)
- Automatic server stop before restore, restart after
- ZIP integrity validation pre-extraction
- World validation post-extraction (level.dat + region/)
- Rollback to .old world directory on validation failure
- Progress callbacks with UI progress bar
- Dashboard backup status indicator

**Requirements:** BACK-01, BACK-02, BACK-03, BACK-04, BACK-05
**Files:** `src/stores/backupStore.ts`, `src/services/backupService.ts`, `src/components/ui/BackupCard.tsx`, `app/backup/index.tsx`

### Phase 6: Nested Config Editor

**Goal:** Replace flat YAML editor with recursive tree view supporting nested objects and arrays.
**Depends on:** Phase 4 (plugin config management)
**Plans:** 1 plan

- [x] 06-01: Nested YAML Config Editor

**Details:**
- `ConfigTreeEditor` + `YamlNode` recursive component architecture
- Expandable/collapsible tree view with ▶/▼ toggles
- Inline scalar editing (string, number, boolean, null) with numeric keyboard for numbers
- Array operations: add items (modal), remove, reorder (up/down arrows)
- Object operations: add keys (modal prompt), remove
- YAML validation on save; errors block save until corrected
- js-yaml round-trip structure preservation (comments excluded)
- Integrated into plugin detail screen

**Requirements:** CONF-01, CONF-02, CONF-03, CONF-04
**Files:** `src/components/ui/ConfigTreeEditor.tsx`, `src/components/ui/YamlNode.tsx`

### Phase 7: Plugin Metadata

**Goal:** Extract and display plugin name, version, author from JAR files; handle corrupted JARs gracefully.
**Depends on:** Phase 4 (plugin management)
**Plans:** 1 plan

- [x] 07-01: Plugin JAR Metadata Extraction and Display

**Details:**
- `getPluginMetadata()` reads `plugin.yml` from JAR via adm-zip
- MANIFEST.MF fallback for legacy plugins without plugin.yml
- `PluginMetadata` TypeScript interface (name, version, author, description)
- Plugin list displays name/version/author subtext; filename fallback
- Corrupted JAR detection via `isCorruptedJar()`
- Warning icon (⚠️) on corrupted plugins; blocked interactions (toggle, navigate)
- Import-time corruption check with file cleanup
- Detail page shows metadata + corrupted warning banner; disables config editor and reload
- 12 unit tests with adm-zip/ js-yaml mocking

**Requirements:** PLUG-03, PLUG-04, PLUG-05
**Files:** `src/services/pluginConfigManager.ts`, `app/plugins/index.tsx`, `app/plugins/[id].tsx`
