## Objective
Enable PocketHost server mapping to the public internet by switching from Vanilla Server JAR to PaperMC and introducing the Playit.gg Spigot plugin as a core downloaded asset.

### <task>
<name>Regex match logs for Playit claim and allocation</name>
<read_first>
- src/services/serverManager.ts
</read_first>
<action>
Modify `emitter.addListener('onLog', ...)` inside `initializeEventListeners`. 
After `useServerStore.getState().actions.addLog(event.log);`:
- Check if `event.log` includes `https://playit.gg/claim/`. If so, extract the claim link using a regex like `(https:\/\/playit\.gg\/claim\/[a-zA-Z0-9]+)` and call `useServerStore.getState().actions.setPlayitProperty('playitClaimUrl', match[1])`.
- Check if `event.log` includes `tunnel has been created` or `successfully setup tunnel`. Because playit.gg output formats can vary slightly, detect general tunnel assignment lines in default plugin logging. In playit 0.2.14, the log usually says `tunnel running, connected at: [IP:PORT]`. Alternatively, we can use a wide regex capturing IPs, but for safety: 
Use match: `const ipMatch = event.log.match(/connected at:\s*([a-zA-Z0-9.-]+:[0-9]+)/i);`
If an IP match is found, call `useServerStore.getState().actions.setPlayitProperty('playitAddress', ipMatch[1])`.
</action>
<acceptance_criteria>
- `src/services/serverManager.ts` contains a regex implementation for `playit.gg/claim/`.
- `playitAddress` is properly extracted and updated.
</acceptance_criteria>
</task>
