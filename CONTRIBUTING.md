# Contributing to PocketHost

Thank you for your interest in improving PocketHost. This document outlines how to contribute and the terms under which contributions are accepted.

## How to Contribute

1. **Check the Roadmap** — Before starting work, review [.planning/ROADMAP.md](.planning/ROADMAP.md) and open issues to avoid duplicate effort.
2. **Open an Issue** — For bugs or features, open an issue using the appropriate GitHub template.
3. **Fork & Branch** — Create a feature branch from `main`.
4. **Follow Conventions** — Adhere to the project's coding standards documented in [.planning/codebase/CONVENTIONS.md](.planning/codebase/CONVENTIONS.md).
5. **Test Your Changes** — Run `npm run typecheck` and verify on an Android device or emulator.
6. **Submit a Pull Request** — Reference the related issue and describe what changed and why.

## Development Setup

```bash
npm install
npm run start        # Expo development server
npm run typecheck    # TypeScript validation
```
